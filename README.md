### Who am I?

I'm a Flutter dev who builds stuff that runs in production — not just demos.

I've worked on apps used by real people, dealt with real payment failures, real race conditions, real backend bills. My thing is building complete systems: from the database schema to the pixel on screen.

I don't just write code — I think about what breaks at 2am when no one's looking.

**What I actually care about:**
- Systems that don't fall apart under load
- Payment flows that handle refunds gracefully
- Code that a future-me won't hate
- Shipping features, not just PRs
